# domain.model

Propósito técnico

Agrupa las subcarpetas del modelo de dominio: `aggregates`, `commands`, `entities`, `events`, `queries` y `valueobjects`.

Qué parte del enunciado resuelve

- Estructura lógica del modelo de dominio que implementa las reglas y los objetos necesarios para gestionar `Equipment`.

Archivos que deberían ir aquí

- Subpaquetes con sus clases correspondientes (ver carpetas específicas).

Significado en DDD (técnico)

Organiza el modelo en componentes con responsabilidades claras para facilitar mantenimiento y evolución.

Explicación para no programadores

Es una forma ordenada de agrupar las piezas que describen qué es una máquina y cómo se trabaja con ella dentro del sistema.
